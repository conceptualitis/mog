Using Mog

Setup

<script>

var Demo = Mog.extend({
  initialize: function () {
    this.sync();
  }
});

var demo = new Demo("demo");

</script>

Mog does a lot of things for you while providing a powerful, event-driven platform to expand upon.

To get started with Mog all we have to do is pass Mog.extend() an object containing an initialize function. This simply gives Mog some code to run when you use the new keyword.

When we pass the string "demo" to new Demo() and call sync() in our initialize function, Mog looks through your HTML and generates a model.

You'll see how Mog takes "demo" and uses it to generate an object.

Moving one value

Mog moves input values into HTML.

Input

Output

Bob

HTML

<input type="text" data-mog-input="demo[name]" value="Bob">
…
<span data-mog-output="demo[name]">Bob</span>

The data-mog-input attribute tells Mog that this HTML element generates a property value. The first part of the value, demo, tells Mog to listen to this element when we call sync(). The second piece, [name], tells Mog which property the HTML element is associated with. In this case, Bob.

The data-mog-output behaves basically the same way, but tells Mog to place values into the element instead of retrieve values from it.

Moving many values

Mog moves many input values into many HTML elements. Any Mog input has this ability.

<strong data-mog-output="demo[name-x]">Dave</strong>
<em data-mog-output="demo[name-x]">Dave</em>

Output

HTML
